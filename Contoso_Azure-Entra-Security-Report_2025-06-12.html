Implement mandatory MFA for all users (currently low adoption) ‚ñ∂
+8 points
Why This Matters

Multi-Factor Authentication (MFA) is your strongest defense against credential-based attacks. Even if passwords are compromised, MFA prevents unauthorized access.

How to Fix

1. Navigate to Azure Portal > Azure Active Directory > Security > Conditional Access

2. Create a new policy requiring MFA for all users

3. Test with pilot group first, then roll out organization-wide

New-MgIdentityConditionalAccessPolicy -DisplayName "Require MFA for All Users"
HIGH
Assign break-glass admin to Exchange Administrator role ‚ñ∂
+8 points
Why This Matters

Break-glass admin accounts ensure you can maintain access to critical services during emergencies or when primary admins are unavailable.

How to Fix

1. Create a dedicated break-glass account (e.g., breakglass@domain.com)

2. Assign it to Exchange Administrator role

3. Store credentials securely and document emergency procedures

New-MgDirectoryRoleMember -DirectoryRoleId (Get-MgDirectoryRole -Filter "DisplayName eq 'Exchange Administrator'").Id -Id (Get-MgUser -Filter "DisplayName eq 'BreakGlass'").Id
MED
Review 6 Global Administrator assignments ‚ñ∂
+4 points
Why This Matters

Too many Global Administrators increases your attack surface. Each additional GA represents a potential security risk if compromised.

How to Fix

1. Review current Global Administrator assignments

2. Assign users to least-privilege roles instead (User Admin, Security Admin, etc.)

3. Keep only 2-4 Global Administrators for break-glass scenarios

Get-MgDirectoryRoleMember -DirectoryRoleId (Get-MgDirectoryRole -Filter "DisplayName eq 'Global Administrator'").Id
MED
Enable Security Defaults or expand Conditional Access policies ‚ñ∂
+4 points
Why This Matters

Security Defaults provide baseline security for tenants without Conditional Access policies. They enforce MFA and block legacy authentication.

How to Fix

1. If you have Conditional Access policies, ensure they cover MFA requirements

2. Otherwise, enable Security Defaults in Azure Portal > Azure Active Directory > Properties

Update-MgPolicyIdentitySecurityDefaultEnforcementPolicy -IsEnabled $true

Restrict guest user permissions ‚ñ∂
+4 points
Why This Matters

Guest users with excessive permissions can create security risks and compliance issues in your tenant.

How to Fix

1. Navigate to Azure Portal > Azure Active Directory > External Identities > External collaboration settings

2. Restrict guest user permissions to "Guest users have limited access to properties and memberships"

3. Review and update guest user access policies

Update-MgPolicyAuthorizationPolicy -AllowedToCreateApps $false -AllowedToCreateSecurityGroups $false
